## 4. 核心差異化

### 4.1 自然語言操作（核心賣點 #1）

```
王董打字：「今天工廠狀況」
↓
IntentClassifier 分類 → ProductionAgent
↓
工具呼叫：query_today_summary
↓
回答：今日工單 12 張（進行中 8）、低於安全庫存 3 項、預計出貨 5 筆 95%
```

**對比**：傳統 ERP 要點 7 個選單、開 3 個報表才能拿到一樣的資訊。

### 4.2 行動優先設計

| 設計準則 | 落地 |
|---|---|
| 行動優先 | 5 tab 大按鈕、最少打字 |
| 3 步操作 | 任何功能 ≤ 3 tap 完成 |
| 離線降級 | AsyncStorage 快取 token + 基本資料 |
| 大字體 | 老闆 / 廠長戶外可讀 |
| 單手操作 | 重要按鈕在拇指自然範圍 |

### 4.3 外協 LINE 協同（業界首創）

```
主廠 → 透過 LINE Bot 發圖紙 + 工序 + 一次性 QR token
       ↓
外協廠老師傅（不註冊任何系統）→ LINE 收到 → 掃 QR
       ↓
回報「完成 80%」+ 拍照
       ↓
主廠工單自動更新 / EventBus 觸發品檢通知
```

**業界痛點**：傳統做法要求外協廠註冊 ERP / 學新軟體，老師傅根本不用。
**我們的解法**：外協廠**不需註冊**、不需學任何新東西，會用 LINE 就會用。

### 4.4 MESH 多廠資料主權

```
HQ（總部）── WireGuard VPN ──┐
                                ├── 主廠 (Factory A) :8001
                                ├── 鍍鋅外協 (Factory B) :8002
                                ├── 檢驗外協 (Factory C) :8003
                                └── 第 N 個外協

HQ 問「全廠 M6 庫存？」 →
  • HQ 平行呼叫 N 廠 /api/factory/mesh/query
  • 每廠只回「該廠合計數字」（不送原始 row）
  • HQ 聚合：3000 + 1500 + 800 = 5300

★ 重點：HQ 永遠看不到「哪批料、誰買的」
★ 整合測試 5/5 PASS — 不是 stub，是真的可運作
```

## 6. 部署選項

### 6.1 四種部署模式

```
模式 A：單機 Docker（最簡單，適合 50 人廠）
├── HQ 後端 + 桌機 UI + DB 全裝在一台
└── 一鍵安裝 install.bat / install.sh

模式 B：雲端託管（推薦給沒有 IT 的廠）
├── 我們代管在 GCP/AWS/Azure
├── 客戶只看到一個 https://your-company.llm-erp.app
└── 月費包含主機 + 備份 + 升級

模式 C：客戶自架（資料主權需求）
├── 客戶買虛擬機 / 自有伺服器
├── 我們提供 Docker image + 部署手冊
└── 客戶 IT 維護

模式 D：MESH 多廠（多廠老闆 / 集團）
├── HQ + N 個 Factory Node
├── WireGuard VPN 串連
├── 每廠資料留在自己廠內
└── HQ 只拿聚合數字
```

### 6.2 安裝時間

| 模式 | 首次安裝時間 | 客戶需做的事 |
|---|---|---|
| A 單機 Docker | **10 分鐘** | 雙擊 install.bat |
| B 雲端託管 | **次日上線** | 填表 + 付款 |
| C 自架 | 0.5-1 天 | 跟我們的工程師遠端協作 |
| D MESH | 1-3 天 / 廠 | 每廠裝一台 + 設 VPN |

## 8. 安全與合規

### 8.1 安全設計

| 層 | 機制 |
|---|---|
| **驗證** | JWT (HS256) + bcrypt 密碼雜湊 |
| **授權** | RBAC 5 層（Tenant→User→Role→Permission→Row-Level Scope）|
| **審計** | AuditMiddleware 全請求記錄 + DecisionLog（AI 決策） |
| **傳輸** | HTTPS（搭配 nginx / Cloudflare Tunnel） |
| **多廠** | WireGuard VPN + 廠級 VLAN |
| **Secret** | `.env` + 建議搭配 HashiCorp Vault（企業版） |
| **Demo bypass** | JWT_SECRET 為預設值才啟用，正式部署自動關閉 |

### 8.2 資料主權

- **MESH 多廠模式**：原始 row **物理上**留在各廠 SQLite/PostgreSQL
- HQ 只能透過 `/api/factory/aggregate` 拿聚合數字
- 整合測試實證：aggregate response 不含 `created_at` / `qty_available` / `items` / `rows` 任一原始欄位

### 8.3 合規預備

| 合規 | 狀態 |
|---|---|
| GDPR（個資刪除權） | 預留 API endpoint，需客戶啟用 |
| ISO 27001 | 文件齊全度 70%，需第三方稽核 |
| 中國個資法 | 可部署在中國機房（DeepSeek/Ollama） |
| HIPAA | 醫療業需客製化加密 |

## 11. 採購 FAQ

### Q1：為什麼比 SAP / Oracle 便宜這麼多？

**因為我們不收授權費**。SAP 那 200 萬主要是「授權 license」+「導入顧問」。我們是開源核心 + 訂閱制（年費），客戶買的是訂閱 + 服務，不是「擁有軟體」。

### Q2：開源會不會風險？

不會。LLM-ERP 本身是商業專案，**開源核心**讓客戶可以：① 自己看 code ② 隨時離開不被綁定 ③ 找其他人接手維護。**我們的價值在服務**，不在鎖死客戶。

### Q3：我們資料安全嗎？

四層保護：
1. **架構層**：MESH 多廠模式資料留在各廠
2. **傳輸層**：HTTPS + WireGuard VPN
3. **儲存層**：bcrypt 密碼 + 可加密磁碟
4. **稽核層**：AuditMiddleware 全紀錄 + DecisionLog

可選擇用 Ollama 本地 LLM **資料完全不外流**。

### Q4：可以客製化嗎？

可以。三種方式：
- **配置**：透過 admin UI 改設定（免費）
- **擴充**：寫新 domain / agent / tool（顧問費）
- **修改核心**：開源所以可以改，建議 fork 維護（自負風險）

### Q5：如果你們公司倒了怎麼辦？

開源核心仍可運作；資料在客戶手上不受影響。可以找任何 Python 工程師接手維護（普及度高的技術棧）。MESH 模式廠內資料完全自有，更穩。

### Q6：付款方式？

- 年付（標準）
- 季付（加價 10%）
- 一次性 license（限自架模式，價格 = 3 年訂閱）

### Q7：可以先試用嗎？

可以。**Demo 模式 14 天免費**，免註冊：
1. 雙擊 `install.bat`
2. 點「Demo 模式」按鈕
3. 用範例資料試完整功能

### Q8：訓練教材有嗎？

齊全：
- 操作手冊（中英）

- 系統架構圖（中英）

| **行動端體驗** | 弱（傳統 UI） | 中（responsive） | **強（原生 App）** |
| **AI 自然語言操作** | 無 | 無 | **核心功能** |
| **LINE Bot 整合** | 客製 | 客製 | **內建** |
| **外協 LINE 協同** | 無 | 無 | **業界首創** |
| **資料主權（多廠）** | 集中式 | 集中式 | **MESH 多廠** |
| **客製能力** | 低（封閉） | 中（開源） | **高（開源 + AI）** |

### 12.2 為什麼不直接用 Excel？

| 場景 | Excel | LLM-ERP |
|---|---|---|
| 老闆問「今天狀況」 | 翻 5 個 sheet 找 | 一句話就回 |
| 業務查庫存 | 打開總機台電腦 | 手機 3 秒 |
| 庫存即時同步 | 不可能 | SSE 推播 |
| 外協廠協同 | 群組 LINE 一團亂 | QR + 自動回填 |
| 稽核軌跡 | 沒有 | AuditMiddleware 全紀錄 |
