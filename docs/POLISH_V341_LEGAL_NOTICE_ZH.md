## 1. 寄 PDF Email 給客戶（P5）⚖️ **法律最敏感**

### 1.1 行為

- LLM 講「寄 QUO-001 給 ABC」→ 出 ConfirmCard
- 確認後產生 PDF + 透過 SMTP 寄附件給客戶 contact_email
- 預設 TTL 15 分鐘（比一般 30 分鐘更短 — 強調慎重）

### 1.2 法律風險（**極重要**）

**對外 Email 之 PDF 可能構成法律意義之**「**要約**」「**承諾**」「**告知**」：

- 🔴 **報價單**：可能構成「要約之邀請」(invitation to treat)；若客戶誤解為「要約」並承諾 → 契約成立之爭議
- 🔴 **銷售訂單**：可能構成「承諾」(acceptance)；客戶可主張價格 / 規格 / 交期已確定
- 🔴 **出貨單**：物權移轉之依據；錯誤寄送 = 帳目失控
- 🔴 **發票相關文件**：須符合**統一發票使用辦法**，erpilot 之 PDF **不是**統一發票

### 1.3 寄錯收件人之風險

- 🔴 **客戶資料外洩**（個資法第 27 條）
- 🔴 **商業機密外洩**（營業秘密法）
- 🔴 **競爭對手取得議價情報**
- 🔴 **客戶對客戶**之誤導（A 客戶看到 B 客戶之價格）

### 1.4 客戶內控義務（**強制**）

導入此功能**前**必須：

- [ ] **指派**對外 Email 之「**最終覆核人**」（通常為業務主管 / 老闆）
- [ ] **限制**`sales.order.update` 權限給經授權人員
- [ ] **建立**「寄送前清單覆核」SOP（收件人 / 主旨 / 附件正確）
- [ ] **設定**SMTP 之**寄件回條**追蹤
- [ ] **訂定**寄錯後之「**召回 / 道歉**」流程
- [ ] **教育**員工：**寄送即不可收回**（除非有對方 IT 配合）

### 1.5 SMTP 配置之安全要求

- ❌ **不要**使用個人 Gmail / Yahoo（容易被誤判垃圾郵件）
- ✅ **應使用**公司網域之 SMTP（如 `noreply@yourcompany.com.tw`）
- ✅ **配置 SPF / DKIM / DMARC** 提升送達率 + 防偽
- ✅ 開啟**寄送 audit log**（誰 / 何時 / 寄給誰 / 寄什麼）

### 1.6 erpilot 之責任邊界

於適用法律所允許之最大範圍內：
- erpilot **提供**寄送工具，**不**驗證 PDF 內容之**正確性**
- erpilot **不**驗證收件人之**真實性**
- erpilot **不**自動處理「召回 email」
- erpilot **不**承擔因寄錯收件人 / 寄錯內容所致之**任何**契約爭議、個資外洩、商業損失

### 1.7 SMTP 未設定時之 dry-run

- 若 `SMTP_HOST` / `SMTP_FROM` 未設定 → 自動 dry-run（不真寄）
- dry-run 模式**僅**顯示預覽，**不**會洩漏資料
- 但客戶**不應**以為 dry-run = 無風險 — 真實寄送前須先測試

## 2. 客戶毛利率分析（P1）⚖️ **議價策略敏感**

### 2.1 行為

- LLM 講「哪 5 個客戶最賺」「ABC 賺不賺錢」
- 系統計算 (revenue − cost) ÷ revenue × 100%
- 列出 top N 並標 🟢🟡🔴 等級

### 2.2 法律 / 商業敏感

毛利率資料屬於：
- **公司內部議價策略**（核心商業機密）
- 若外流給**員工**：可能用於跳槽攜帶資訊（違反**保密義務**）
- 若外流給**客戶 A**：客戶 A 可主張「同業 B 賺比較少 → 我也要降價」
- 若外流給**競爭對手**：等同**送情報**

### 2.3 客戶內控義務

- 限制 `sales.order.read` 權限**僅給**經授權之主管
- 不得**截圖 / 列印 / 匯出**毛利率分析
- 員工**簽署保密協議**含「毛利率屬於營業秘密」之明確條款
- 對員工**內訓**時強調「即使在私下對話也不可洩漏」

### 2.4 計算方法之限制

- 成本以 `Product.standard_cost` 為準 — **不**含人工、製造費用、行銷費
- 不含**退貨 / 折讓**之沖銷
- 不含**呆帳損失**
- 不含**機會成本**（產能被低毛利客戶佔用之損失）
- 客戶**正式財報分析**應由**會計師**用更完整的方法

### 2.5 erpilot 免責

erpilot 提供之毛利率為**啟發式分析**，**不**取代正式財務分析；erpilot 對因依賴此分析所做之**議價決策、客戶取捨**所致之損失，**不承擔責任**。

## 3. 訂單跟單（P2）

### 3.1 行為

- 講「QUO-001 後續變成什麼」→ 顯示 Quote → SO → 出貨 → 應收 → 收款 timeline

### 3.2 風險

- timeline 含**契約執行細節**
- 若給**競爭對手**：洩漏交易結構
- 若給**客戶**（看自己的訂單可能 OK，但其他客戶不行）：個資 / 商業機密

### 3.3 客戶責任

- 限制權限：`sales.order.read`
- 對外提及具體交易**須**取得對方同意

## 4. 資料健康檢查（P8）

### 4.1 行為

- 檢查重複客戶 / 重複料件 / BOM 循環 / 孤兒庫存 / 無 email 客戶 / 無成本料件
- 顯示 🔴🟠🟡🟢 嚴重度

### 4.2 限制

- 健康檢查為**啟發式偵測**
- 結果不**取代**正式 IT 稽核 / 內控查核
- 客戶之**特殊業務規則**不在此工具偵測範圍

### 4.3 客戶責任

- 健康檢查發現之問題**由 IT / 內控主管**處理
- **不應**讓非 IT 員工直接處理（可能造成更大資料污染）

## 5. Chat Thumbs Feedback（P7）

### 5.1 行為

- 使用者按 👍 / 👎 → 寫到 AuditLog（action="chat.feedback"）

### 5.2 員工監督邊界

- Feedback 含**員工對系統的滿意度** → 可能用於員工績效評估
- 客戶應**告知員工**：此 feedback 為**改善系統用**，**不**用於人事考核
- 若用於人事考核，須**事前告知**並取得同意

### 5.3 個資處理

- Feedback 之 `comment` 欄位可能含**自由文字**（員工可能寫敏感資訊）
- 客戶須**定期審視**並**遮罩**敏感欄位
- 保存期應與**員工資料**一致

## 7. 案例：常見錯誤使用模式

### 7.1 ❌ 不該做

- **寄 PDF 給錯誤客戶**（如 A 客戶之報價單寄到 B 客戶）
- 把毛利率分析**截圖**傳到 LINE 群組
- 用毛利率資料作為**員工 KPI**之依據（員工會找客戶吵架）
- 跟客戶**口頭提及**其他客戶之 timeline
- 在沒設定 SPF/DKIM 之狀況下大量寄客戶 email（容易被當垃圾郵件）

### 7.2 ✅ 應該做

- 寄 email 前**人工覆核**收件人 + 附件
- 毛利率分析**僅在主管會議**討論，**不**留紙本
- 用毛利率分析作為**內部議價策略**，**不**外流
- 訂單跟單僅**內部追蹤**用
- SMTP **使用公司網域** + SPF/DKIM/DMARC 配置

## 10. 國際法 / 跨國使用警告

| 地區 | 額外法律 | 對 v3.41 影響 |
|---|---|---|
| 🇪🇺 EU | GDPR Art. 6 / 32 | Email 寄送須**合法基礎**；客戶資料**最小化** |
| 🇺🇸 US (CAN-SPAM) | CAN-SPAM Act | Email **必須含 opt-out 連結** + 公司地址 |
| 🇺🇸 US (CA) | CCPA | 客戶有「**不被販售**」權利 — 毛利率分析不可用於對外議價 |
| 🇨🇳 China | PIPL | Email 跨境寄送須**單獨同意** |
| 🇯🇵 Japan | APPI | 客戶資料用途**變更**需通知 |
| 🇸🇬 SG | PDPA | DNC 名單對行銷 email 有限制 |

⚠️ **erpilot 之 SMTP 寄送不**自動加上 unsubscribe link / 公司地址 — 客戶須**自行**符合上述規定。
