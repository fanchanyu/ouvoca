## 1. 系統簡介

### 1.1 LLM-ERP 是什麼？

LLM-ERP 是一套**為 50-100 人小型製造業設計**的 AI 原生 ERP（企業資源規劃）系統。

**核心特色**：
- 🗣️ **自然語言操作**：用 LINE 或手機講話就能查資料、下指令
- 📱 **行動優先**：手機、平板、桌機都好用
- 🔗 **外協協同**：外協廠用 LINE 掃 QR 不用註冊
- 🌐 **MESH 多廠**：主廠 + 外協廠資料即時同步，但各自獨立
- 🛡️ **角色權限**：業務只看自己客戶、外協只看自己工單

### 1.2 適合誰用？

| 角色 | 怎麼用 |
|---|---|
| 👔 **老闆（王董）** | LINE 問「今天狀況」→ AI 回答 |
| 👨‍💼 **業務（小陳）** | 手機在客戶面前查庫存、報價 |
| 👨‍🏭 **廠長（林廠長）** | 手機看工單進度、收推播 |
| 👩‍💻 **採購倉管（阿玲）** | 桌機建 PO、手機掃 QR 盤點 |
| 👴 **外協廠（老吳）** | LINE 掃 QR 回報完工，不用註冊 |

## 2. 快速開始

### 2.1 啟動系統

如果是 IT 部署，請參考 [部署指南 (DEPLOYMENT.md)](./DEPLOYMENT.md)。
如果是試用者，跟著以下三步：

```bash
# 步驟 1：啟動全棧
docker compose up -d --build

# 步驟 2：載入示範資料
docker compose exec backend python -m scripts.seed

# 步驟 3：開瀏覽器
# 桌面 UI:    http://localhost:5173
# War Room:   http://localhost:8080
# API 文件:   http://localhost:8000/docs
```

### 2.2 三種登入方式

| 方式 | 帳號 | 適用 |
|---|---|---|
| **正式登入** | `admin` / `admin123` | 完整功能體驗 |
| **Demo 模式** | 按「以 Demo 模式進入」 | 不註冊快速體驗 |
| **未來：LINE 綁定** | 加 ERP 官方帳號 | 老闆專用 |

> 💡 **Demo 模式**：適合快速試用。一旦正式部署設定 `JWT_SECRET`，此模式會自動關閉。

### 2.3 載入行業範例資料

不同行業有不同的示範資料，您可以選擇接近的：

```bash
# 金屬加工（CNC 螺絲螺帽）
docker compose exec backend python -m scripts.seed_industries metal

# 塑膠射出（射出成型）
docker compose exec backend python -m scripts.seed_industries plastic

# PCB 電子組裝
docker compose exec backend python -m scripts.seed_industries pcb

# 食品加工（烘焙）
docker compose exec backend python -m scripts.seed_industries food

# 紡織印染
docker compose exec backend python -m scripts.seed_industries textile

# 全部都載入（5 個行業並存）
docker compose exec backend python -m scripts.seed_industries all
```

## 3. 登入與語言切換

### 3.1 登入頁

打開 `http://localhost:5173`，您會看到漸層動態的登入畫面：

1. **右上角**：🇹🇼 繁中 / 🇺🇸 EN 切換

系統自動偵測您的瀏覽器語言。若想手動切換：

| 位置 | 操作 |
|---|---|
| 登入頁右上角 | 點 🇹🇼 / 🇺🇸 |
| 進入系統後右上角 | 點地球圖示 → 選擇語言 |

**切換是即時的**——不需要重新登入。系統會記住您的偏好。

## 5. 各角色操作指南

### 5.1 老闆（王董）— 用 LINE 管整個工廠

#### 早上 7:00：收每日摘要

開 LINE → ERP 官方帳號 → 自動推：

```
早安王董！📈

✅ 昨日出貨 12 單（$580K）
⚠️ 今日有 2 張急單待釋放
🔴 M6 螺絲剩 300 個（低於安全庫存）

[查看詳情] [處理急單]
```

#### 任何時候：問問題

```
您：今天工廠表現如何？

Bot：
今日重點摘要：
✓ 進行中工單 15 張，平均進度 68%
⚠️ 待釋放工單 3 張（最急 SO-20260514-003）
✓ 銷售 12 筆，營收 $580,000
🔴 M6 螺絲缺料，建議補貨 2000 個
```

> 💡 **小技巧**：用自然語言，不用學系統術語。Bot 會自動翻譯。

#### 桌面 UI 操作

打開 `http://localhost:5173`，登入後看到 **Dashboard 儀表板**：

| 元素 | 說明 |
|---|---|
| **AI 智能摘要** | 一句話告訴您今天最重要的事 |
| **4 個關鍵卡片** | 營收 · 工單 · 採購 · 庫存警示 |
| **最近工單** | 進度條一目了然 |
| **庫存警示** | 低於安全的零件清單 |

每個卡片都**可點擊**跳到詳情。

### 5.2 業務（小陳）— 手機在客戶面前查資料

#### 場景：客戶問「下批可交多少？」

1. 開手機 App
2. 點 💬 AI 助手
3. 對麥克風講：「客戶 A 的 PRD-GEAR-A 6 月可承諾交期」
4. 3 秒後 Bot 回：「可承諾 500 台，最快交期 6/15」

#### 場景：客戶問「上次給我的價是多少？」

1. AI 助手輸入：「客戶 A 的 PRD-GEAR-A 歷史單價」
2. Bot 回：「最近 3 次：5/12 $4500、4/20 $4400、3/15 $4300」

> 💡 **業務看不到別的業務的客戶**——這是 own scope 保護。
> 系統自動限制「業務 A 只能看自己建的客戶」。

---

### 5.3 廠長（林廠長）— 手機看推播

#### 設定推播

1. 桌面 UI 進入 **🔑 我的權限**
2. 角色裡確認有 `plant_manager`
3. 訂閱事件：`stock.below_safety` / `wo.completed` / `nc.created`

#### 收到警示時

LINE / App 會跳出：
```
⚠️ 工單延遲警示

WO-20260514-002（軸承座 B）
原預計：5/20
目前進度：30%（落後 2 天）

[查看詳情] [指派緊急處理]
```

點開直接進手機 App 工單頁。

---

### 5.4 採購倉管（阿玲）— 手機掃 QR

#### 場景：收料

1. 倉庫拿到供應商送來的料
2. 開手機 App → 「收料」
3. 掃 PO 上的 QR 碼
4. 系統自動帶出資訊：
   ```
   PO-20260514-001
   大華精密工業
   M6-BOLT-20 × 1000
   ```
5. 輸入實收數量「1000」
6. 拍照存證
7. 按「確認」→ 自動：
   - 更新庫存 +1000
   - PO 標記為「已收貨」
   - LINE 通知林廠長「料到了」

#### 場景：盤點

1. 開手機 App → 「盤點」
2. 選擇儲位
3. 掃儲位 QR
4. 輸入實際數量
5. 拍照（如有差異）
6. 系統自動：
   - 計算差異
   - 若差異 > 5% 自動標記異常
   - 通知主管 review

### 5.5 外協廠（老吳）— LINE 掃 QR 不用註冊

#### 老吳的故事

老吳是鍍鋅外協廠老闆，只有他和兩個徒弟。
他**不會用電腦**，也不想註冊新系統。

#### 完工回報流程

1. 主廠列印「外協派工單」交給老吳，上面有 QR
2. 老吳完工後，打開 LINE
3. 加主廠 ERP 官方帳號
4. 點選單 → 「回報外協」
5. Bot：「請掃描派工單 QR」
6. 老吳掃 QR
7. Bot 顯示：
   ```
   工單 OS-20260514-003
   品名：M6 螺絲鍍鋅
   派工：500 件
   請輸入完工數量
   ```
8. 老吳回：「500」
9. Bot：「請拍張完工照片」
10. 老吳拍照上傳
11. Bot：「✅ 已回報，謝謝吳老闆！」
12. 主廠林廠長立刻收到 LINE 通知

**全程不用註冊、不用學新軟體、不用打字打很多**。

## 6. AI 助手使用法

### 6.1 怎麼問

用**自然語言**，不用學系統術語：

| ✅ 好問法 | ❌ 不要這樣問 |
|---|---|
| 庫存還剩多少 M6 螺絲？ | execute SELECT FROM parts WHERE... |
| 進行中的工單有幾張？ | List all WO with status=in_progress |
| 哪個客戶最近欠錢？ | Query AR table |

### 6.2 AI 能做什麼

| 模組 | 範例問題 |
|---|---|
| 📦 庫存 | 「庫存最少的 5 個料是什麼？」「M6 螺絲還剩多少？」 |
| 🛒 採購 | 「大華精密最近的 PO 狀態」「上次跟誰買的 SUS304？」 |
| 🏭 生產 | 「今天進行中工單」「WO-001 進度」 |
| 💰 銷售 | 「A 客戶歷史單價」「本月銷售總額」 |
| 🔬 品質 | 「最近不良品」「本月合格率」 |
| 💳 會計 | 「逾期應收清單」「本月傳票」 |

### 6.3 AI 的限制

- **目前不能寫資料**（v2.0），只能查詢
- **複雜問題會跟你確認**：例如「下個月排程」會問「您指的是 MPS 還是工單排程？」
- **不會的問題會誠實說**：「我目前不確定，您可以查 X 頁面」

### 6.4 沒有 LLM API Key 怎麼辦？

AI 助手會顯示：「[demo 模式] 偵測到意圖 = xxx，但尚未設定 LLM_API_KEY」。

**這時系統其他 11 個頁面都正常運作**——只是 AI 對話功能停用。
等 IT 設定好 API Key 後（DeepSeek / OpenAI / Anthropic 擇一），就會自動恢復。

## 7. 常見問題

### Q1：為什麼某個按鈕灰掉？

→ 您的角色沒有該操作的權限。請聯絡 IT 或老闆增加權限。
查看 **🔑 我的權限** 確認您有什麼。

### Q2：為什麼看不到別的業務的客戶？

→ **這是設計**，不是 bug。每個業務只看自己建立的客戶（own scope），避免跳槽帶走客戶。

### Q3：手機 App 在哪下載？

→ v2.0 還是用瀏覽器版（http://localhost:5173），響應式設計已優化手機。
原生 App 在 Phase 1 規劃中。

### Q4：可以同時用中文和英文嗎？

→ 個人偏好可獨立切換。同一個系統，你看中文，外國同事看英文。
資料本身（料號名稱、客戶名）以您建立時輸入的文字為準。

### Q5：忘記密碼怎麼辦？

→ v2.0 請聯絡 IT 重置（後台改 hashed_password）。
Phase 2 會加自助密碼重置（email 驗證）。

### Q6：怎麼匯出資料？

→ 各頁面右上角有 **匯出 CSV** 按鈕（Phase 2 規劃中）。
目前可用 API：`GET /api/<domain>/<resource>` 取得 JSON。

### Q7：可以多個工廠用同一套系統嗎？

→ **可以**。這是 MESH 多廠設計的核心：
- HQ（主廠）+ N 個 Factory Node
- 每個廠資料獨立、不外流
- HQ 可跨廠查詢聚合
- 設定方式請見 [DEPLOYMENT.md](./DEPLOYMENT.md) §4

### Q8：客戶外協廠多家共用怎麼辦？

→ 同一個外協廠（如老吳鍍鋅）可服務多個主廠。
每個主廠列印的 QR 派工單有獨立 token，老吳掃時 Bot 會自動辨識是哪家。

## 9. 疑難排解

### 9.1 進不去登入頁

| 症狀 | 解法 |
|---|---|
| 「無法連線到後端」 | 確認 backend 是否啟動：`docker compose ps` |
| 503 / 504 | 重啟：`docker compose restart backend` |
| 401 不停彈出 | 清 localStorage：F12 → Application → Clear |

### 9.2 看不到資料

| 症狀 | 解法 |
|---|---|
| 列表全空 | 確認已 seed：`docker compose exec backend python -m scripts.seed` |
| 只看到自己的 | 這是 row-level 設計（own scope），不是 bug |
| 401 forbidden | 您的角色沒此權限，找 IT 加 |

### 9.3 AI 助手不回應

| 症狀 | 解法 |
|---|---|
| 「demo 模式」 | 設定 `.env` 的 `LLM_API_KEY` |
| 「LLM 呼叫失敗」 | 確認 API Key 有效、有額度 |
| 等很久 | 可能是 LLM 在思考，最多等 60 秒 |

### 9.4 手機顯示異常

| 症狀 | 解法 |
|---|---|
| 字太小 | 雙指放大或設定瀏覽器字級 |
| 排版破版 | 確認您用主流瀏覽器（Chrome / Safari） |
| 觸控無反應 | 重新整理頁面 |

## 10. 進階：管理員專用

### 10.1 開新員工帳號

```
1. 進入 桌面 UI / 系統 / 權限管理（需 admin 權限）
2. 系統 → 組織 → 員工 → 新增
3. 填入：員工編號 / 姓名 / Email / 部門
4. 系統 → 組織 → 使用者 → 新增
5. 帳號 / 預設密碼 / 關聯員工
6. 進入 權限管理 → 為該員工指派角色（如 sales_rep）
```

### 10.2 臨時授權

例：王董出差，林廠長代理 3 天權限

```
桌面 UI → 權限管理 → 找到 boss 角色 → 複製 →
建立 boss_temp_2026_05_20 → 指派給林廠長 →
設定到期日 2026-05-22 23:59 → 儲存
```

到期後自動失效。

### 10.3 看誰改了什麼

```
桌面 UI → 系統 → 稽核紀錄（Phase 1.5 規劃中）
目前可從 backend 查 permission_audit 表
```

## 13. Mobile App 使用（NEW in v2.3）

### 13.1 安裝與啟動

Phase 1 已完成 Expo 行動 App 骨架（5 個 tab）。

**前置**：Node.js 18+、與後端在同一個 Wi-Fi。

```bash
cd frontend-mobile
npm install
# 編輯 app.json 中的 extra.apiBaseUrl 改為您電腦的內網 IP
# 範例：http://192.168.1.100:8000
npm start
```

從 Terminal 看到 QR Code 後：
- iOS：相機掃 QR → 自動開 Expo Go App
- Android：Expo Go App 掃 QR

### 13.2 5 個 Tab 操作

| Tab | 功能 |
|---|---|

## 14. MESH 多廠協同（NEW in v2.3）

### 14.1 啟用場景

當您有 **2 個以上工廠 / 外協廠** 想串連，但**不想資料集中**：
- 主廠 + 鍍鋅外協 + 檢驗外協（典型 3 廠）
- 集團多個生產基地
- 客戶要求「我們資料留在自己廠內」

### 14.2 啟動 Factory Node

每個工廠的伺服器跑：

```bash
FACTORY_ID=plating  FACTORY_NAME='鍍鋅外協' \
PORT=8002  HQ_URL=http://hq.your-domain.com:8000 \
python backend/factory_node.py
```

廠端會自動 register 給 HQ，HQ 之後可以聚合查詢。

### 14.3 HQ 端跨廠查詢

```bash
# 看哪些廠註冊
curl http://hq:8000/api/factory/list

# 跨廠聚合 M6 螺絲庫存
curl -X POST 'http://hq:8000/api/factory/aggregate?domain=inventory&part_no=M6'

# 回應：{ total: 4500, per_factory: {main: 3000, plating: 1500}, ... }
```

### 14.4 資料主權保證

`/api/factory/aggregate` 的回應**只包含聚合數字**，不含任何原始 row（已實證 5 個整合測試）：
- ❌ 不送 created_at / qty_available / items / rows 任一欄位
- ✅ 只送 total / per_factory_total / responded_count
