## 1. 系統簡介

### 1.1 LLM-ERP 是什麼？

LLM-ERP 是一套**為 50-100 人小型製造業設計**的「對話式 ERP（企業資源規劃）」系統。

「ERP」翻成白話就是：**幫公司管庫存、管訂單、管採購、管生產、管報表**的軟體。
傳統 ERP（像 SAP、Oracle）要花幾百萬買、員工受訓 1-3 個月才會用。

**LLM-ERP 把這件事翻轉**：
你不用學介面，你用「講話」（打字）就能操作整套系統。

### 1.2 三大核心承諾

| 承諾 | 白話翻譯 |
|---|---|
| **🗣️ 自然語言操作** | 你想做什麼，就打字告訴 AI，它會幫你做。不用學選單在哪裡。 |
| **🛡️ 全程確認 + 90 秒撤銷** | 任何會動到資料的操作（下單/改單/刪單），AI 都會跳「確認卡」讓你按。按錯了 90 秒內可以撤銷。 |
| **⚡ 即時、不等月底** | 老闆隨時問「今天狀況」，10 秒內拿到答案，不用等月報。 |

### 1.3 適合誰用？

這套系統設計給**「50-100 人小型製造業」**的四種角色：

| 角色 | 主要工作 | 用什麼裝置 |
|---|---|---|
| 👔 **老闆（王董）** | 隨時掌握公司狀況、看財務數字 | 辦公室桌機 Chrome 瀏覽器 |
| 👨‍💼 **業務（小陳）** | 在客戶面前秒答庫存、報價、交期 | 筆電（外出帶 Chrome 連 VPN） |
| 👨‍🏭 **廠長（林廠長）** | 看哪裡卡關、釋放工單、調整排程 | 辦公室桌機 + 廠房大屏 |
| 👩‍💻 **採購兼倉管（阿玲）** | 下採購單、收料、盤點 | 桌機 + USB 條碼槍 |

### 1.4 你能用它做什麼？

四類操作（業界術語叫「CRUD」）：

| 簡寫 | 中文 | 你會講的話舉例 |
|---|---|---|
| **C** = Create | 增 | 「跟長江廠下 100 個 M6 螺絲」 |
| **R** = Read | 查 | 「列出庫存低於安全庫存的零件」 |
| **U** = Update | 改 | 「把 SO-2025-0042 的交期改到 6/10」 |
| **D** = Delete | 刪 / 取消 | 「取消 PO-2025-0107」 |

## 2. 第一次登入

### 2.1 開啟系統

請 IT 把系統部署好之後（如果還沒，請看 `INSTALLATION_ZH.md`），你只要做這一步：

1. 打開 **Chrome 瀏覽器**（推薦最新版；Edge / Firefox 也可以）
2. 在網址列輸入：`http://localhost:5173`（IT 給你的網址）
3. 按 Enter

你會看到登入畫面：

│         │ └──────────────────┘ │         │

│         │ └──────────────────┘ │         │
│         │                       │         │

│         │                       │         │
│         │  ── 或 ──             │         │

### 2.2 輸入帳號密碼

| 步驟 | 操作 |
|---|---|
| 1 | 滑鼠**點「帳號」欄位** |
| 2 | 鍵盤打字輸入帳號（首次預設：`admin`） |
| 3 | 用 Tab 鍵或滑鼠**點到「密碼」欄位** |
| 4 | 鍵盤輸入密碼（首次預設：`admin123`） |
| 5 | 滑鼠**點藍色「登入」按鈕**（或按 Enter） |

> ⚠️ **第一次登入後請立刻改密碼**！
> 進入後：右上角頭像 → 個人設定 → 修改密碼

### 2.3 不想記帳號？用 Demo 模式

如果你只是想試試看，不想記帳號：
- 滑鼠**點下方「以 Demo 模式進入」按鈕**
- 就會直接進入系統，所有功能都能玩

> ⚠️ Demo 模式上線後會被 IT 關掉。正式上線一定要用真實帳號。

### 2.4 切換語言

登入畫面**右上角**有兩個國旗按鈕：

| 圖示 | 意思 |
|---|---|
| 🇹🇼 | 切換到繁體中文 |
| 🇺🇸 | 切換到 English |

登入後一樣在**右上角**可以切換。**切換是即時的**，不用重新登入。

### 3.5 ⚙️ 設定頁 (Settings) ✨ v3.13 新增

點左側 sidebar 的 **「⚙️ 設定」** 進到設定頁，有 4 個區塊（**所有電腦小白會用到的東西都在這頁**）：

#### A. 🤖 AI 助手設定

| 步驟 | 操作 |

## 4. 對 AI 講話：4 種 CRUD 操作

**這是整本手冊最重要的章節，請仔細看完。**

ERP 系統能做的事，總結就 4 類：「**查 / 增 / 改 / 刪**」。
我們把這 4 類都做成「用打字就能完成」。下面每一類給你 1-2 個實例。

### 4.1 「查」（Read）— 不會跳確認卡，直接給答案

## 5. ConfirmCard 確認卡

### 5.1 為什麼一定要有 ConfirmCard？

**因為 AI 會「幻覺」**（業界術語叫 Hallucination）。

什麼意思？AI 偶爾會「自己想像」資料。例如：
- 你說「跟長江廠下單」，AI 可能搞混成「跟長榮廠下單」
- 你說「100 個」，AI 可能聽成「1000 個」
- 你說「M6 螺絲」，但庫存裡有 M6-BOLT、M6-NUT、M6-WASHER 三種

如果沒有 ConfirmCard，AI 直接下單 → **你公司可能多花 10 倍的錢，或下錯廠**。

**ConfirmCard 的設計哲學**：
> AI 負責理解你的話、產生草稿；
> **人類負責最後一眼確認**，按下「確認」才真的執行。

### 5.2 ConfirmCard 上有什麼？

每張 ConfirmCard 都有：

| 區塊 | 內容 |
|---|---|
| **標題** | 要做什麼動作（建立 PO / 修改 SO / 取消 WO ...） |
| **摘要表格** | 所有欄位的對照（從哪解析出來的、值是多少） |
| **影響說明** | 這個動作會引起什麼連鎖反應 |
| **3 個按鈕** | ✓ 確認 / ✏ 修改 / ✗ 取消 |
| **倒數計時** | 90 秒不點 → 自動取消，避免你忘了 |

### 5.3 ConfirmCard 的 4 種風險等級

系統內部把工具分 4 級，不同風險顯示不同顏色：

| 等級 | 顏色 | 範例 | ConfirmCard 行為 |
|---|---|---|---|
| 🟢 **查（Read）** | 無卡片 | 「列出庫存」 | 直接給答案，不跳卡 |
| 🟡 **軟寫（Soft-write）** | 黃色 | 「儲存搜尋條件」 | 簡單卡片 + 簡單 Undo |
| 🟠 **硬寫（Hard-write）** | 橘色 | 「建立 PO」「改交期」 | **完整 ConfirmCard + 90 秒 Undo** |
| 🔴 **高危（Critical）** | 紅色 | 「刪客戶」「取消大額 PO」 | **紅色卡片 + 要打字確認** |

## 6. Slot-filling 反問機制

### 6.1 什麼是 Slot-filling？

英文「Slot」就是「欄位」。如果你話講得不完整，AI **不會自己編**，會反問你。

### 6.2 實例：欄位不完整時的對話

你輸入：

```
給中鋼下 PO
```

AI 回答：

```
好的，要建立採購單給中鋼。請問：

1. 要買什麼料？（料號或品名）
2. 數量多少？
3. 交期什麼時候？

也可以一次回答，例如：「M6-BOLT-20，1000 個，下週五」
```

你回：

```
M6 螺絲 1000 個下週五
```

AI 回答（部分還沒解析清楚）：

```
「M6 螺絲」對應到 3 個料號，請選一個：

   ┌─────────────────────────────────────┐
   │ ⓪ M6-BOLT-20（六角螺絲，最常用）     │
   │ ① M6-NUT（M6 螺帽）                  │
   │ ② M6-WASHER（M6 墊片）              │
   └─────────────────────────────────────┘

請輸入編號（例如「0」）或料號全名。
```

你回 `0` → AI 跳出完整 ConfirmCard。

### 6.3 為什麼要這樣設計？

**不會反問 = AI 亂猜 = 你公司下錯料**。
我們寧可多問你幾句，也不要讓你公司賠錢。

> 💡 **反問上限 3 次**：如果 AI 問 3 次還對不上你的意思，
> 它會說「請改用左側『採購』頁面的表單填寫」，把你帶到傳統介面。

## 7. 90 秒 Undo 撤銷

### 7.1 怎麼撤銷？

任何「增/改/刪」操作後，**90 秒內**你可以撤銷。

#### 方法 1：用講的

在 AI 對話框輸入：

```
撤銷上一個動作
```

或

```
取消剛才那筆
```

AI 回答：

```
✅ 已撤銷 PO-2026-0142 的建立
   採購單已標記為「已撤回」
   長江精密工業已收到撤回通知
```

#### 方法 2：點按鈕

執行完操作後，AI 回應下方會出現一個小按鈕：

```
✅ 已建立採購單 PO-2026-0142

   [ ↩ 撤銷（剩 67 秒）]
```

直接點它就好。

### 7.2 撤銷的限制

| 情況 | 能撤嗎？ |
|---|---|
| 90 秒內，採購單還沒收貨 | ✅ 可以 |
| 90 秒內，採購單已部分收貨 | ❌ 不行（已產生實體流程） |
| 超過 90 秒 | ❌ 請走正常「取消單據」流程 |
| 別人按確認的單，你想撤 | ❌ 只能撤自己按的 |

## 9. 三軌授權

LLM-ERP 採用三軌授權，**選對的省幾百萬**。

| 軌道 | 適合誰 | 費用 | 重點 |
|---|---|---|---|
| 🟢 **AGPL** | 想看原始碼、不介意分享自己修改 | 免費 | 完全開源 |
| 🌱 **Small Business** | 50-100 人廠、≤ 20 個同時上線的人 | **完全免費** | **含閉源 connector**（鼎新/正航接口） |
| 🔵 **Commercial** | > 20 人同時上線、ISV、SaaS 業者 | 議價 | 解除 AGPL 條款限制 |

### 怎麼申請 Small Business 軌？

完全免費，但要登記。請看：

- 📄 **`LICENSE-COMMERCIAL.md`**（在程式根目錄）
- 📄 **`docs/COMMERCIAL_LICENSING_FAQ_ZH.md`**

寄信給授權窗口，附上「公司名 + 員工數 + 預期同時上線人數」就會收到 Small Business 條款的書面授權。

## 10. 常見問題

### Q1：我打字打錯字 AI 會看得懂嗎？

→ 大多看得懂。「M6 螺絲」「m6 buoy 」「m6 螺絲」AI 都會嘗試對應。
看不懂時 AI 會反問你。

### Q2：AI 回答錯了怎麼辦？

→ 如果是「查」回答錯：再問一次，把問題說清楚。
如果是「增/改/刪」的 ConfirmCard 內容不對：點「✏ 修改」或「✗ 取消」，不要按確認。

### Q3：對話歷史會保留多久？

→ 預設 30 天（IT 可調）。
要看歷史：右上角 👤 頭像 → 「我的對話紀錄」。

### Q4：可以切換角色嗎？例如業務想看廠長視角？

→ 一個帳號對應一個角色，**不能自己切**。
要看別人的視角請請 IT/老闆在「⚙️ 設定 → 使用者權限」加你的角色。

### Q5：我可以匯出 Excel 嗎？

→ 列表頁右上角有「📥 匯出 CSV」按鈕（CSV 可用 Excel 打開）。
Excel `.xlsx` 直出排在 Phase 2。

### Q6：忘記密碼怎麼辦？

→ 找 IT 重設。自助密碼重設（Email 連結）排在 Phase 2。

### Q7：可以用語音輸入嗎？

→ **Phase 4 才會做**（Whisper 語音轉文字）。
目前只能打字。

### Q8：可以多廠共用一套系統嗎？

→ 可以。叫做「MESH 多廠」，每個廠資料獨立但 HQ 可以聚合查詢。
詳細請看 `NETWORK_DEPLOYMENT_ZH.md`。

### Q9：AI 不回應怎麼辦？

→ 看 §11 疑難排解 → 11.3。

### Q10：商業使用前要做什麼？

→ 看第 9 章「三軌授權」+ `LICENSE-COMMERCIAL.md`。

| 症狀 | 怎麼解 |
|---|---|
| 瀏覽器顯示「無法連線到此網站」 | 確認 IT 已啟動後端（`docker compose ps`） |
| 503 / 504 錯誤 | 請 IT 重啟：`docker compose restart backend` |
| 401 一直跳出來 | 按 F12 → Application → Storage → Clear site data，重新整理 |

### 11.2 看不到資料

| 症狀 | 怎麼解 |
|---|---|
| 列表全空 | IT 還沒載入示範資料（`docker compose exec backend python -m scripts.seed`） |
| 只看到自己建立的客戶 | **這是設計**，不是 bug。RBAC 保護你的客戶不被別人看見 |
| 點按鈕跳 403 forbidden | 你的角色沒這個權限，請 IT 加角色 |

### 11.3 AI 助手不回應

| 症狀 | 怎麼解 |
|---|---|
| 跳出「🤖 AI 助手還沒啟用」3 步驟卡 | **不用找 IT**！自己進 ⚙️ 設定 → AI 設定區，按引導申請貼 key（看 §3.5 + [`HOW_TO_GET_LLM_API_KEY_ZH.md`](./HOW_TO_GET_LLM_API_KEY_ZH.md)）|
| 右上角顯示「⚠️ AI 未設定」 | 同上，點 ⚠️ 自動跳到設定頁 |
| 顯示「API key 無效」/「quota 用完」 | 進 ⚠️ 設定頁更新 key（不需 IT 改 .env）|
| 等很久（超過 30 秒） | LLM 在思考中，最多等 60 秒；超過請 IT 看 log |
| AI 一直問同一個問題 | 反問 3 次仍對不上 → AI 會建議你改用左側表單頁面 |
| **不知道這頁怎麼用** | 點**右下角 💡 浮球**問 AI「這頁怎麼用？」（§3.7）|

### 11.4 ConfirmCard 異常

| 症狀 | 怎麼解 |
|---|---|
| 倒數結束自動取消了 | 重新對 AI 講一次就好 |
| 點「✏ 修改」沒反應 | 重新整理頁面（F5），再對 AI 講一次 |
| 點「✓ 確認」後一直轉圈 | 看右下角是否有錯誤訊息；找 IT 看後端 log |

### 11.5 撤銷失敗

| 症狀 | 怎麼解 |
|---|---|
| 「超過 90 秒，不能撤銷」 | 請走正式「取消單據」流程 |
| 「該動作已被下游流程使用」 | 例如 PO 已部分收料，要先做退料才能撤 |
| 「您不是執行者，不能撤銷」 | 請當初按確認的人來撤 |

### 11.6 還是不行？

聯絡內部 IT，並提供：
- 截圖（按 Win+Shift+S 框選）
- 你輸入的指令
- 錯誤訊息（如有）
- 時間（幾點幾分）

或寫 Email 到貴公司 ERP 維運窗口。
