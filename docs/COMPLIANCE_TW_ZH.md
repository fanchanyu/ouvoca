## 1. 稅務合規

### 1.1 營業稅（5%）

| 項目 | 狀態 | 對應功能 |
|---|---|---|
| 401 一般營業稅申報書 | ✅ 完整支援 | `GET /api/tax/tw/401?year=2026&period_no=1` |
| 403 進銷項憑證明細 | ✅ 完整支援 | `GET /api/tax/tw/403?direction=sales` |
| 405 零稅率銷售額 | 🟡 部分（需設客戶為「外銷」）| 待 Phase 1.5 |
| 雙月期申報 | ✅ 內建 6 期 helper | period_no 1-6 |
| 統一發票號碼字軌管理 | ✅ 內建 | 隨機碼 + 字軌驗證 |
| 統一編號驗證（含檢查碼）| ✅ 內建 | `validate_tax_id()` |

### 1.2 申報期程

| 期別 | 月份 | 申報截止 |
|---|---|---|
| 第 1 期 | 1-2 月 | 3/15 |
| 第 2 期 | 3-4 月 | 5/15 |
| 第 3 期 | 5-6 月 | 7/15 |
| 第 4 期 | 7-8 月 | 9/15 |
| 第 5 期 | 9-10 月 | 11/15 |
| 第 6 期 | 11-12 月 | 隔年 1/15 |

LLM-ERP 在每期截止前 5 天自動推送 LINE 提醒（規劃中）。

### 1.3 營利事業所得稅（25%/20%）

| 項目 | 狀態 |
|---|---|
| 月度損益試算 | ✅ `GET /api/analytics/gross-margin` |
| 年度結算暫繳 | 🟡 需手動匯出 |
| 結算申報書（401-1）| ❌ 需顧問搭配 |

## 2. 電子發票

### 2.1 法規對應

| 法規 | 我們的支援 |
|---|---|
| 統一發票使用辦法 第 7 條 | ✅ EInvoice 結構符合 |
| 財政部 MIG 3.2.1 | ✅ `to_mig_dict()` 輸出對齊 |
| 電子發票 B2B / B2C | ✅ 都支援（buyer_tax_id 可空） |
| 載具歸戶（手機條碼 3J0002）| ✅ carrier_type / carrier_id 欄位 |
| 自然人憑證條碼（EJ0113）| ✅ 同上 |
| 捐贈碼 | ✅ npo_id 欄位 |
| 折讓單 | 🟡 cancel API 已有，折讓單格式待補 |
| 作廢 / 撤銷 | ✅ `cancel(invoice_no, reason)` |

### 2.2 加值中心對接

我們提供 **Adapter 介面** + Mock provider，真正接 production 加值中心要：

```python
# 在 backend/app/integrations/einvoice_tw.py 替換 default_provider
class GuanTradeProvider(EInvoiceProvider):
    """關貿網路 — 大型加值中心。"""
    def submit(self, inv): ...
    def cancel(self, invoice_no, reason): ...
```

支援的加值中心（規劃接入）：
- 關貿網路（GVB）
- 中華電信 HiCloud
- 凱基綠界（綠界科技）
- 永豐 e-Pay

### 2.3 自我驗證內建項目

- ✅ 統一編號 8 碼檢查碼（演算法）+ 全相同數字排除
- ✅ 發票號碼格式（AB12345678 / AB-12345678）
- ✅ 稅額計算（5%）+ 四捨五入
- ✅ 總額 = 銷售額 + 稅額（±1 元誤差）

## 3. 個資法 / 資料保護

| 個資法條文 | 狀態 |
|---|---|
| 第 6 條（特種個資）| 🟡 不收集（建議），需客戶自行避免存入 |
| 第 8 條（告知義務）| ❌ 需客戶提供隱私權聲明 |
| 第 11 條（更新刪除）| ✅ 客戶 / 員工模型有 soft delete |
| 第 12 條（資安）| ✅ bcrypt 密碼 / RBAC / Audit |
| 第 27 條（個資外洩通報）| 🟡 AuditMiddleware 可追溯 |
| 個資清冊 | 🟡 列出收集欄位（見下） |

### 3.1 收集的個人資料項目

| 對象 | 欄位 |
|---|---|
| **員工** | 姓名 / Email / 電話 / 部門 / 職稱 / 雇用日 |
| **客戶聯絡人** | 姓名 / Email / 電話 |
| **供應商聯絡人** | 姓名 / Email / 電話 |
| **使用者** | 帳號 / hash 後密碼 / 最後登入時間 |

**未收集**：身分證字號 / 健保卡號 / 銀行帳戶 / 信用卡（NCC 規範外）

### 3.2 GDPR 對齊（外銷國際品牌時）

| 權利 | 我們的對應 |
|---|---|
| Right to access | `GET /api/auth/me` |
| Right to erasure | 預留 API（需客戶啟用） |
| Right to portability | 匯出 CSV / xlsx 內建 |
| Data localization | MESH 多廠模式 — 資料留在各廠 |

## 4. 勞動法 / 二代健保

| 項目 | 狀態 |
|---|---|
| 員工資料 | ✅ Employee 模型 |
| 出勤打卡 | 🟡 需第三方整合（Phase 2+）|
| 薪資計算 | ❌ 不做（建議用 8thlife / 102 等） |
| 二代健保扣繳（額外補充保費）| ❌ 不做 |
| 勞工退休金 6% 提繳 | ❌ 不做 |

> 設計選擇：**HR 不深入薪資**，定位是「ERP + AI」不是「人資系統」。
> 提供 API 給第三方薪資系統串接。

## 6. 合規等級對照

| 合規等級 | LLM-ERP 達成度 | 需客戶補做 |
|---|---|---|
| **5 人以下個體戶** | 100% ✅ | 無 |
| **50-100 人小型製造業（ICP）** | 90% ✅ | 串外部薪資系統 |
| **跨國集團（500+ 人）** | 60% 🟡 | + 集團合併報表 / 多幣別 / 多公司 |
| **上市櫃公司** | 40% 🟠 | + 重大訊息 / IFRS / 內控報表 |
| **金融業 / 醫療業** | 不適合 ❌ | 需業界專用系統 |

## 7. 自行設定清單

導入時需客戶自行設定 / 確認的項目：

### 公司基本資料
- [ ] 公司統一編號
- [ ] 公司名稱（中英文）
- [ ] 公司地址（中英文）
- [ ] 負責人姓名 / 統一發票字軌
- [ ] 營業項目代號
- [ ] 主管機關（國稅局分局）

### 稅務設定
- [ ] 適用稅率（一般 5% / 零稅率 / 免稅）
- [ ] 兩月期申報 vs 月申報（一般為兩月）
- [ ] 是否使用電子發票
- [ ] 加值中心廠商

### 銀行 / 金流
- [ ] 公司銀行帳戶（用於 AR 收款 / AP 付款記載）
- [ ] 金流廠商（綠界 / 街口 / LinePay …，若有）

### 法定隱私
- [ ] 個資告知聲明（給員工 / 客戶簽署）
- [ ] 客戶 / 供應商 / 員工同意書
