## 1. 為什麼建議先確認授權

### 1.1 商用 ERP 常見的授權模式

下列為市場上常見之授權模式，**具體適用何種模式以及對「第三方系統介接」之規範，依各廠商之合約條款而異**——請以您與原 ERP 廠商之**官方授權文件 / 合約**為準：

| 常見授權模式 | 說明 | 與「第三方系統介接」可能之關聯 |
|---|---|---|
| **Per-Named-User（具名使用者授權）** | 以個別具名使用者數計費 | 部分廠商之合約對「以服務帳號介接」可能有額外規範 |
| **Concurrent User（同時使用者授權）** | 以同時在線數計費 | 第三方系統之持續連線可能佔用同時使用者名額 |
| **Module License（模組授權）** | 以購買之功能模組計費 | 跨模組之資料存取可能涉及未購買模組之範圍 |
| **API / Integration License（API / 整合授權）** | 視合約而定為單獨計費項目 | 部分廠商之 API 屬另購項目 |
| **ODBC / 外接資料庫授權** | 視合約而定為單獨計費項目 | 部分廠商對 DB 直連可能有額外授權項目 |
| **Read-Only / Read-Replica（唯讀 / 副本授權）** | 視合約而定為單獨計費項目 | 即使僅為唯讀，仍可能需依合約取得適當授權 |

> 💡 上表僅為一般性介紹，**不代表任何特定廠商必然採用何種模式或必然有何限制**。任何疑問請逕洽該 ERP 廠商之經銷商或法務窗口。

### 1.2 合規稽核可能透過下列管道發現

軟體合規稽核之觸發與發現管道，常見者包括（不限於）：

- **DB 端 audit log**：許多商用 ERP 於 DB 層保留連線記錄
- **License Manager / Telemetry**：部分 ERP 內建授權監控機制
- **年度或定期合規稽核**：依您與廠商或經銷商之合約安排
- **DB 連線 metadata**：service account 名稱、connection string 等可被分析

> ⚠️ 建議於啟用任何外部連線前，先依您與原 ERP 廠商之合約條款處理授權事宜。

## 2. 啟用 connector 前的建議準備

### 2.1 ✅ Step 1 — 與原 ERP 廠商確認授權範圍

建議與您原 ERP 廠商 / 經銷商**以書面**確認下列事項（內容依貴司實際使用情境調整）：

> **本公司計畫由第三方系統（Ouvoca）以共用 / 服務帳號讀取本 ERP 之資料庫或 API，使用範圍包含 [SELECT / 部分 UPDATE / Schema Mapping 等]。請貴司確認本使用方式是否在現行授權範圍內，如否，需如何補足。**

可接受之書面形式（依貴司法務政策決定）：
- 原 ERP 廠商之正式回函
- 合約增補附件
- 經銷商之書面確認
- 電子郵件回覆（建議保留完整 thread）

> 💡 **建議**：書面確認較口頭承諾於日後爭議解決時更具參考價值。

### 2.2 ✅ Step 2 — 視原廠回覆購買必要授權

依原廠回覆，可能需要購買之附加項目（**具體項目、是否需要、價格均依各廠商報價而定**）：

| 可能之附加項目 | 適用情境 |
|---|---|
| Integration License | 第三方系統介接 |
| ODBC License | DB 直連模式 |
| Service Account License | 共用 / 服務帳號 |
| API License | REST API / SOAP 模式 |
| Read-Replica License | 唯讀副本 / 報表 DB |

> 💡 Ouvoca 提供之 connector 為**技術元件**，**不含**任何第三方 ERP 之授權費用。具體授權費依貴司與該廠商之報價或合約。

### 2.3 ✅ Step 3 — 留存相關文件

建議將下列文件依貴司文件管理政策保存：
- 原 ERP 廠商之授權確認文件
- 附加授權之合約 / 發票
- Ouvoca connector 啟用之內部簽核紀錄

> 💡 完善之文件管理有助於日後合規查核或爭議解決。

## 4. 常見問答（FAQ）

### Q1：我們是小公司（30 人），是否仍需注意？
**A**：合規責任與公司規模無直接關係，建議依您與原 ERP 廠商之合約條款處理。若有疑慮，請逕洽該廠商或您的法律顧問。

### Q2：我們只用 Ouvoca 讀（SELECT），不會改原 DB，是否仍需確認授權？
**A**：建議仍與原廠確認。許多授權合約之適用範圍包含「存取」行為，並不僅限於「寫入」。具體適用情形請依合約條款。

### Q3：我請我們 IT 同事的個人帳號讓 Ouvoca 用，這樣可以嗎？
**A**：此種做法在技術上可行，但**建議審慎評估**：
- 部分授權合約對「個人帳號之使用範圍」有規範，請以合約條款為準
- 該員工離職時 Ouvoca 將失去連線能力
- 個人帳號權限通常較廣，從資安最佳實務而言，建議使用「最小權限服務帳號」
- 建議與貴司法務 / 原 ERP 廠商確認該方式是否在授權範圍內

### Q4：原廠回覆「不確定」「沒有對應的授權項目」怎麼辦？
**A**：建議要求**書面回覆**，明確記載原廠對該使用情境之認知與態度。如無法取得明確回覆，建議審慎評估後再決定是否啟用 connector，必要時諮詢法律顧問。

### Q5：我已經啟用了 connector 才看到這份提醒，怎麼辦？
**A**：建議：
1. 視情況考慮暫停連線
2. 留存啟用期間之連線 log
3. 主動與原 ERP 廠商溝通說明，依其指引處理
4. 必要時諮詢法律顧問

### Q6：Ouvoca 可以幫忙與原 ERP 廠商溝通授權嗎？
**A**：Ouvoca 不參與客戶與第三方 ERP 廠商間之溝通。建議由貴司 IT / 法務 / 採購主管直接洽詢原廠或經銷商（對授權條款最為熟悉）。

### Q7：我們是用開源版 / 試用版的 ERP，是否仍適用？
**A**：開源授權（AGPL / GPL 等）通常對第三方系統存取較無限制，但**商業版 / 試用版**之授權條款依各版本而異。請以您所使用版本之**官方授權文件**為準。

### Q8：我們僅做一次性資料遷移後即斷線，是否需確認授權？
**A**：技術上可一次性遷移後斷線。然而「遷移期間之存取」仍屬使用行為之一，建議依 §2 確認授權範圍後再執行。
