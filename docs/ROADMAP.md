### 1.4 驗收劇本（必須跑通）

**劇本 A：王董的早晨**
1. 王董 LINE 加 ERP 官方帳號
2. 系統 push「早安王董，請輸入您的員工編號完成綁定」
3. 王董輸入「E0001」→ 完成綁定
4. 隔天早上 7 點，Bot 自動推「昨日摘要」：出貨 12 單 / $580K、急單 2 張、庫存警示 1 項
5. 王董點「查看詳情」→ Flex Message 展開

**劇本 B：小陳的客戶拜訪**
1. 小陳在客戶端開 LINE
2. 問 ERP Bot「A 客戶 PRD-GEAR-A 歷史單價」
3. Bot 回：「最近 3 次：5/12 $4500、4/20 $4400、3/15 $4300」
4. 客戶問「下月可交多少」
5. 小陳問 Bot「PRD-GEAR-A 6 月可承諾交期」
6. Bot 回 ATP 數量

**劇本 C：林廠長的推播**
1. 林廠長設定「庫存低於安全 + 工單延遲 → 推 LINE」
2. M6-BOLT-20 庫存低於安全
3. 林廠長 LINE 收到推播
4. 點擊推播 → 直接進手機 App 庫存詳情頁

**劇本 D：老吳的外協回報**
1. 廠內生成外協派工單 OS-20260520-001（鍍鋅 500 件）
2. 列印含 QR 的派工單給老吳
3. 老吳完成後，LINE 找 ERP Bot
4. 選「我要回報外協」→ 掃 QR
5. Bot 顯示派工資訊
6. 老吳輸入「500」、拍照
7. Bot 回「✅ 已回報」
8. 廠內林廠長手機收到推播 + 阿玲收到待收貨通知

### 4.A VMI 經典場景

1. HQ 王董 LINE 問「全廠 M6 螺絲庫存」
2. HQ Bot 並發查詢 主廠 / 鍍鋅廠 / 檢驗廠

## 重要原則

1. **Phase 1-4 是 MVP**：必須做完且做好。
2. **Phase 5+ 不主動**：等 5 個客戶有相同需求才開發。
3. **每個 Phase 都要能演示給王董看**：不能演示 = 沒做完。
4. **每個功能都要對應 persona**：找不到 persona = 不做。
5. **客戶反饋 > 我們的想像**：定期訪談客戶，更新優先級。
