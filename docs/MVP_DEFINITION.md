# MVP 範圍定義（MVP DEFINITION）— v3.0

> **MVP 哲學**：少做、做精、做對。
> 50-100 人廠最痛的 6 件事 → 做到極致；其餘暫緩。
> 任何超出 MVP 範圍的功能必須有客戶實際反饋才開發。

> ⚡ **v3.0 戰略軸轉**：v2 的 8 大功能（含 LINE Bot / Mobile / 外協）收斂為 **6 大功能（純桌機 Chat）**。

---

| # | 功能 | 解決誰的痛 | 上線後馬上能做什麼 | 對應 Phase |
|---|---|---|---|---|

## 2. MVP 不做（Out of Scope）

### v3.0 移除（從 v2 砍）

| 功能 | 為什麼砍 | 復活條件 |
|---|---|---|
| **LINE Bot Webhook + 老闆儀表板** | 桌機 Chat 同樣解決王董需求；LINE Bot 是不同平台、不同 SDK，不值得分散精力 | 客戶簽約後明確要求 |
| **Mobile App（Expo）** | 桌機 Chat 涵蓋 80% 場景；行動端複雜度高（推播/FCM/上架）成本高 | 客戶簽約後明確要求 |
| **外協 QR 派工單 + LINE 回報** | 沒有外協廠 persona 後不需要此功能 | 客戶簽約後明確要求 |
| **Speech-to-Text 行動端** | 桌機 Chat 用 Whisper 即可，行動端 STT 砍掉 | Phase 4 桌機 Whisper |

### 暫緩到 Phase 5+（先看客戶反饋再決定）

| 功能 | 為什麼暫緩 | 復活條件 |
|---|---|---|
| **APS 演算法引擎（GA/SA/TS）** | 50-100 人廠老師傅憑經驗排得比演算法快 | 客戶廠成長到 200+ 人才需要 |
| **S&OP 月度產銷協調** | 小廠通常一週開一次會就夠 | 客戶實際要求才做 |
| **多階審批工作流** | 小廠老闆一聲令下就過了 | 50+ 客戶要求 |
| **複雜成本會計** | 小廠多用「年底盤」模式 | 客戶上市櫃需求 |
| **月結鎖期** | 太重，會計用 Excel 對更快 | 大客戶會計合規需求 |
| **多階 BOM 遞迴展開（>3 階）** | 小廠 BOM 多在 2-3 階以內 | 客戶實際有 4+ 階 BOM |
| **RCCP/CRP 細產能規劃** | 小廠機台少，目視管理夠用 | 客戶要求 OEE 分析 |
| **What-if 多情境模擬** | 小廠憑感覺決策 | 客戶要求科學決策支援 |
| **SOC2 / GDPR 合規** | 小廠用不到 | 接到歐美客戶要求 |
| **演算法選擇器** | 太學術 | 永遠不做（除非變學術專案） |

### 永遠不做（明確刪除）

| 功能 | 原因 |
|---|---|
| **大型企業財務模組（IFRS/合併報表）** | 小廠用不到 |
| **HR 薪資計算** | 一般小廠用 104 人資、出勤打卡機 |
| **CRM 行銷自動化** | 小廠靠老闆人脈 |
| **EDI（電子資料交換）** | 大型供應鏈才用 |
| **MES 全工序自動化** | 小廠多手動 |

## 4. MVP 完成的判準（Definition of Done）— v3.0

### 4.1 技術判準

- [ ] 50 人廠跑系統 1 個月，**至少 30 人/日**有使用
- [ ] **AI 對話操作 > 70%**（取代 70% 的傳統介面點擊）
- [ ] **CRUD 4 種操作 AI 都能做**（查/增/改/刪 4/4）
- [ ] hard-write 操作 **誤操作率 < 5%**（被 Undo 比例）
- [ ] API p95 latency **< 200ms**
- [ ] 0 個 P1 級 bug（系統當機、資料遺失）

### 4.2 商業判準

- [ ] 客戶說「**這比 Excel 方便**」
- [ ] 客戶說「**新員工 2 小時就會用**」
- [ ] 客戶說「**不用學系統就能下單**」
- [ ] 客戶願意付 **30-50 萬/年**
- [ ] 客戶願意 **推薦給同業**（NPS > 30）

### 4.3 體驗判準

- [ ] 新員工 **2 小時內** 學會使用
- [ ] 老闆 **打字一句話** 就拿到報表
- [ ] 採購 **30 秒** 就能送出一張 PO
- [ ] 改錯了 **90 秒內** 可以 Undo

## 6. 給開發者的 MVP 紀律（v3.0）

### 每寫程式前的「五問」

1. **這對 MVP 6 大功能有貢獻嗎？** 沒有就不寫。
2. **這 50-100 人廠用得到嗎？** 用不到就降級。
3. **這能用一句話對 Chat 講出來嗎？** 不能就補 tool / slot。
4. **這是 hard-write 嗎？有 ConfirmCard 嗎？** 沒有就加。
5. **這超過 3 步操作嗎？** 是就簡化。

### 每完成功能後的「三檢」

1. **能演示給王董看嗎？** 不能就還沒做完。
2. **能在 Chat 一句話觸發嗎？**（若適用）不能就還沒做完。
3. **hard-write 有 ConfirmCard 嗎？** 沒有就還沒做完。
